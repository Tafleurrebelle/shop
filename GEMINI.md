# Обзор проекта

Это проект сайта-витрины или портфолио, созданный с помощью генератора статических сайтов **MkDocs** и темы **Material for MkDocs**.

Ключевой особенностью является архитектура **"Content-as-Data"**: сайт не требует ручного обновления HTML или конфигурационных файлов при добавлении новых товаров. Вместо этого, JavaScript-модули автоматически обнаруживают новые товары, сканируя папки в `/docs/products/`.

Интерактивность, такая как каталог товаров и корзина, реализована на чистом JavaScript и использует `localStorage` браузера для хранения состояния корзины между сессиями.

## Сборка и запуск

-   **Локальная разработка:**
    Для запуска локального сервера с автоматическим обновлением при изменениях, используй команду:
    ```bash
    mkdocs serve
    ```

-   **Публикация на GitHub Pages:**
    Для сборки сайта и его автоматической публикации на GitHub Pages (в ветку `gh-pages`), используй команду:
    ```bash
    mkdocs gh-deploy
    ```

## Соглашения по разработке

### Добавление нового товара

Процесс добавления нового товара полностью автоматизирован. Чтобы добавить новый товар, выполни следующие шаги:

1.  Создай новую папку в `docs/products/` с последовательным именем (например, `item_00003/`).
2.  Скопируй в нее шаблон `index.md` из любого другого товара.
3.  Создай в ней файл `meta.json`, заполнив все поля: `id`, `name`, `price`, `description`, `image`.
4.  Помести в эту же папку файлы изображений (`image_1.jpg`, `image_2.jpg` и т.д.).

Сайт автоматически обнаружит и отобразит новый товар в каталоге.

### Структура проекта

-   **Исходные файлы:** Весь контент (страницы, товары, изображения) находится в папке `docs/`.
-   **Конфигурация:** Основные настройки сайта, включая навигацию, тему и подключение скриптов, находятся в файле `mkdocs.yml`.
-   **Интерактивность:** Вся динамика сайта управляется JavaScript-модулями, расположенными в `docs/assets/js/`.
-   **Публикация:** Готовый сайт публикуется в ветку `gh-pages` и доступен через GitHub Pages.
